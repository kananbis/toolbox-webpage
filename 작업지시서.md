# 작업지시서: 검색 유입형 무료 웹 유틸리티 사이트 MVP 개발

## 1. 프로젝트 목표

사용자가 검색을 통해 유입되어 바로 사용할 수 있는 간단한 웹 유틸리티 사이트를 개발한다.

초기 목표는 서버 비용을 최소화하기 위해 백엔드, DB, 로그인 기능 없이 동작하는 정적 웹사이트를 만드는 것이다. 각 툴은 독립적인 SEO 페이지를 가지며, 브라우저 내 JavaScript로 계산/변환/생성 기능을 수행한다.

향후 Google AdSense 광고 적용을 고려하여 SEO, 페이지 속도, 모바일 사용성, 개인정보처리방침, 사이트맵 구조를 처음부터 반영한다.

---

## 2. 핵심 요구사항

### 2.1 서비스 형태

* 무료 온라인 유틸리티 모음 사이트
* 회원가입 없음
* 로그인 없음
* DB 없음
* 서버 API 없음
* 모든 계산/변환/생성 기능은 가능하면 브라우저에서 처리
* 정적 페이지 기반
* 모바일 우선 반응형 UI
* SEO 친화적인 페이지 구조

## 7. SEO 요구사항

각 페이지는 다음 메타 정보를 가져야 한다.

```ts
title
description
canonical
og:title
og:description
og:type
og:url
```

### 7.1 SEO 제목 예시

```txt
글자수 세기 - 공백 포함/제외 글자 수 계산기
퍼센트 계산기 - 증가율, 할인율, 비율 계산
날짜 계산기 - 날짜 차이, D-day 계산
비밀번호 생성기 - 안전한 랜덤 비밀번호 만들기
JSON 정리기 - JSON 포맷팅, 압축, 유효성 검사
```

### 7.2 각 툴 페이지 콘텐츠 요구사항

각 툴 페이지에는 기능 UI만 넣지 말고 아래 설명 콘텐츠를 반드시 포함한다.

* 이 도구가 무엇인지
* 어떤 상황에서 쓰는지
* 사용 방법
* 계산 기준 또는 변환 기준
* 예시 2개 이상
* FAQ 3개 이상
* 관련 툴 3개 이상

## 11. 기능별 상세 요구사항

### 11.1 글자수 세기

입력 텍스트를 기준으로 다음을 실시간 계산한다.

* 공백 포함 글자 수
* 공백 제외 글자 수
* 단어 수
* 줄 수
* 문장 수

버튼:

* 복사
* 초기화

---

### 11.2 퍼센트 계산기

다음 계산 모드를 제공한다.

* A의 B%는 얼마인가
* A는 B의 몇 %인가
* A에서 B로 몇 % 증가/감소했는가

입력값 검증을 포함한다.

---

### 11.3 날짜 계산기

다음 기능을 제공한다.

* 시작일과 종료일 사이의 일수 계산
* 시작일 기준 N일 후 날짜
* 시작일 기준 N일 전 날짜

결과는 년/월/일과 총 일수로 표시한다.

---

### 11.4 비밀번호 생성기

옵션:

* 길이
* 대문자 포함
* 소문자 포함
* 숫자 포함
* 특수문자 포함
* 비슷한 문자 제외 옵션

기능:

* 비밀번호 생성
* 복사
* 강도 표시

---

### 11.5 JSON 정리기

기능:

* JSON 포맷팅
* JSON 압축
* JSON 유효성 검사
* 에러 메시지 표시
* 복사

잘못된 JSON 입력 시 앱이 깨지지 않게 예외 처리한다.

---

### 11.6 Base64 변환기

기능:

* 텍스트 → Base64 인코딩
* Base64 → 텍스트 디코딩
* 잘못된 Base64 입력 시 에러 표시
* 복사
* 초기화

## 14. 광고 적용 대비

초기에는 실제 광고 코드를 넣지 않는다.

대신 광고가 들어갈 수 있는 영역만 컴포넌트로 만들어 둔다.

```txt
AdPlaceholder.astro
```

광고 위치:

* 툴 UI 아래
* 본문 중간
* 페이지 하단

단, 툴 사용성을 방해하지 않도록 툴 입력 영역 위에는 광고를 넣지 않는다.

## 17. 개발 순서

### Phase 1: 프로젝트 생성

* Astro 프로젝트 생성
* TypeScript 설정
* Tailwind CSS 설정
* 기본 레이아웃 생성
* Header/Footer 생성
* 글로벌 스타일 정리

### Phase 2: 기본 페이지 구현

* Home
* Tools 목록
* 카테고리 페이지
* About
* Contact
* Privacy
* Terms

### Phase 3: 툴 5개 우선 구현

우선 아래 5개부터 완성한다.

```txt
1. 글자수 세기
2. 퍼센트 계산기
3. 날짜 계산기
4. 비밀번호 생성기
5. JSON 정리기
```

각 페이지는 실제 기능뿐 아니라 SEO 설명 콘텐츠, FAQ, 관련 툴 링크까지 포함한다.

### Phase 4: 나머지 툴 구현

남은 15개 툴을 동일한 템플릿 기반으로 구현한다.

### Phase 5: SEO/성능 정리

* title/description 점검
* sitemap.xml 생성
* robots.txt 생성
* canonical URL 적용
* Open Graph 메타 적용
* Lighthouse 성능 점검
* 모바일 화면 점검

### Phase 6: 배포 준비

* Cloudflare Pages 배포 가능하도록 빌드 확인
* 환경변수 구조 정리
* 사이트 URL 상수화
* contact email 상수화

## 18. 완료 기준

아래 조건을 만족하면 MVP 완료로 본다.

* Astro 프로젝트가 정상 빌드된다.
* `/` 메인 페이지가 있다.
* `/tools` 툴 목록 페이지가 있다.
* 20개 툴 페이지가 존재한다.
* 최소 5개 이상의 툴은 실제 기능이 정상 동작한다.
* 모든 툴 페이지에 SEO title/description이 있다.
* 모든 툴 페이지에 사용 방법/FAQ/관련 툴 영역이 있다.
* About, Contact, Privacy, Terms 페이지가 있다.
* sitemap.xml과 robots.txt가 생성된다.
* 모바일 화면에서 깨지지 않는다.
* 사용자 입력 데이터는 서버로 전송되지 않는다.
* 프로젝트 README에 실행/빌드/배포 방법이 정리되어 있다.

## 20. 주의사항

* 백엔드 서버를 만들지 않는다.
* DB를 만들지 않는다.
* 로그인 기능을 만들지 않는다.
* 복잡한 회원 기능을 넣지 않는다.
* 광고 코드는 초기에는 넣지 않는다.
* 외부 API 의존성을 최소화한다.
* 사용자가 입력한 텍스트를 서버로 전송하지 않는다.
* 기능보다 페이지 속도와 SEO 구조를 우선한다.
* 공통 템플릿과 공통 데이터를 사용해 나중에 툴을 쉽게 추가할 수 있게 한다.

## 21. 최종 목표

초기 MVP는 “완벽한 기능”보다 “SEO 가능한 구조와 확장성”이 중요하다.

따라서 1차 개발의 최종 목표는 다음이다.

```txt
검색 유입이 가능한 정적 유틸리티 사이트의 기본 구조를 만들고,
20개 툴 페이지를 보유하며,
그중 핵심 5개 툴은 실제 기능까지 완성된 상태로 배포 가능하게 만든다.
```
